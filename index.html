Informacion Capa 1 - Modelo OSI

Esta capa se ocupa de la transmision fisica de los datos a traves del medio de red. Se encarga de los aspectos electricos, mecanicos y funcionales de la transmision de datos.

Informacion Capa 2 - Modelo OSI

Esta capa se encarga de la transmision confiable de datos a traves de un enlace fisico. Proporciona la deteccion y correccion de errores, control de flujo y control de acceso al medio.

Informacion Capa 3 - Modelo OSI

Esta capa se encarga del enrutamiento de los paquetes de datos a traves de la red. Se ocupa de establecer, mantener y finalizar las conexiones de red, y tambien de la fragmentacion y reensamblaje de los paquetes.

Informacion Capa 4 - Modelo OSI

La capa de transporte garantiza una comunicacion confiable y eficiente entre los dispositivos finales. Se encarga de dividir los datos en segmentos, proporcionar control de flujo y control de errores, y asegurar que los datos lleguen en el orden correcto.

Informacion Capa 5 - Modelo OSI

La capa de sesion se encarga de establecer, mantener y finalizar las sesiones de comunicacion entre las aplicaciones. Proporciona servicios de dialogo y control de sincronizacion.

Informacion Capa 6 - Modelo OSI

Esta capa se encarga de la representacion y el formato de los datos para garantizar la compatibilidad entre los diferentes sistemas. Se ocupa de la codificacion, compresion y encriptacion de los datos.

Informacion Capa 7 - Modelo OSI

Capa de aplicacion: La capa de aplicacion es la capa mas alta del modelo OSI y se encarga de proporcionar servicios de red a las aplicaciones. Incluye protocolos como HTTP, FTP, SMTP, entre otros.

Informacion Capa 1 - Modelo TCP/IP

Esta capa se encarga de la transmision fisica de los datos a traves del medio de red. Incluye protocolos como Ethernet, Wi-Fi y PPP.

Informacion Capa 2 - Modelo TCP/IP

La capa de Internet maneja el direccionamiento logico y el enrutamiento de los paquetes de datos. El protocolo principal en esta capa es el Protocolo de Internet (IP).

Informacion Capa 3 - Modelo OSI

Esta capa se encarga de la entrega confiable de datos entre dispositivos finales. Los protocolos principales son TCP (Transmission Control Protocol) y UDP (User Datagram Protocol).

Informacion Capa 4 - Modelo OSI

La capa de aplicacion es donde las aplicaciones interactuan con la red. Incluye protocolos como HTTP, FTP, SMTP, DNS, entre otros.

La DeepWeb es una parte de Internet que no esta indexada por los motores de busqueda convencionales y a la que no se puede acceder a traves de navegadores normales. En la DeepWeb se encuentran contenidos que no son publicos y requieren autenticacion para acceder a ellos, como documentos corporativos o bases de datos gubernamentales.

    La DarkWeb, por otro lado, es una red dentro de la DeepWeb que se utiliza para actividades ilegales y anonimas, como el trafico de drogas, la venta de armas, el comercio de informacion robada, entre otros. Para acceder a la DarkWeb se necesita un software especial y el anonimato es clave.

El modelo OSI (Open Systems Interconnection) se compone de 7 capas. A continuacion, se muestra como se implementan en terminos de software y hardware:

    Capas implementadas por el software (sistema operativo u otro software):

  • Capa de aplicacion: Esta capa es implementada principalmente por el software, que proporciona servicios de red a las aplicaciones. Ejemplos de protocolos de esta capa son HTTP, SMTP y FTP.
  • Capa de presentacion: Esta capa se encarga de la representacion y el formato de los datos. El software se encarga de convertir los datos en un formato adecuado para su transmision. Ejemplos de protocolos de esta capa son SSL y TLS.
  • Capa de sesion: Esta capa se encarga de establecer, mantener y finalizar sesiones de comunicacion entre dispositivos. El software gestiona la apertura y cierre de sesiones. Ejemplos de protocolos de esta capa son SIP y NetBIOS.
  • Capa de transporte: Esta capa proporciona la entrega confiable de los datos y se encarga de dividir los datos en segmentos mas pequenos. El software se encarga de controlar el flujo y la secuencia de los segmentos. Ejemplos de protocolos de esta capa son TCP y UDP.

    Capas implementadas por el hardware:

  • Capa de red: Esta capa se encarga del enrutamiento de los datos a traves de la red. El hardware de enrutamiento, como los routers, implementa esta capa. Ejemplos de protocolos de esta capa son IP y ICMP.
  • Capa de enlace de datos: Esta capa se encarga de la transmision de datos entre nodos adyacentes en la red. El hardware de red, como las tarjetas de red, implementa esta capa. Ejemplos de protocolos de esta capa son Ethernet y Wi-Fi.
  • Capa fisica: Esta capa se encarga de la transmision fisica de los bits a traves de los medios de comunicacion. El hardware, como los cables y los transceptores, implementa esta capa.

El funcionamiento del escaneo ARP se basa en el intercambio de mensajes ARP entre los dispositivos de la red. Aqui tienes una descripcion general de como funciona:

  • Funcionamiento del protocolo ARP: Cuando un dispositivo en una red local necesita enviar datos a otro dispositivo, necesita conocer la direccion MAC correspondiente a la direccion IP de destino. El protocolo ARP permite obtener esta informacion.
  • Envio de solicitud ARP: El dispositivo emisor envia una solicitud ARP de difusion (broadcast) a toda la red local, preguntando: "Quien tiene la direccion IP X?". La solicitud ARP contiene la direccion IP del dispositivo emisor y su direccion MAC.
  • Respuesta ARP: El dispositivo que tiene la direccion IP solicitada responde con una respuesta ARP, indicando su direccion MAC correspondiente. Esta respuesta se envia directamente al dispositivo emisor.
  • Registro en la tabla ARP: Una vez que el dispositivo emisor recibe la respuesta ARP, registra la direccion IP y la direccion MAC asociada en su tabla ARP. Esta tabla se utiliza para mantener un registro de las correspondencias entre direcciones IP y direcciones MAC en la red local.
  • Identificacion del objetivo: Primero, se selecciona el dispositivo o sistema que se va a escanear. Puede ser una direccion IP especifica o un rango de direcciones IP.
  • Seleccion de tecnicas de escaneo: Existen diferentes tecnicas de escaneo de puertos disponibles, cada una con su enfoque y nivel de sigilo. Algunas de las tecnicas comunes incluyen: Escaneo TCP: Esta tecnica envia paquetes TCP (Transmission Control Protocol) a traves de los diferentes puertos y analiza las respuestas. Si se recibe una respuesta, significa que el puerto esta abierto y hay un servicio en ejecucion en ese puerto. Escaneo UDP: Similar al escaneo TCP, esta tecnica envia paquetes UDP (User Datagram Protocol) a traves de los puertos y analiza las respuestas. Los puertos que devuelven una respuesta se consideran abiertos. Escaneo de puertos SYN: Esta tecnica utiliza paquetes SYN (Synchronize) para iniciar una conexion con los puertos en el dispositivo objetivo. Si se recibe una respuesta SYN/ACK (Acknowledge), significa que el puerto esta abierto. Escaneo furtivo (stealth scanning): Estas tecnicas, como el escaneo FIN, XMAS o NULL, envian paquetes que no siguen la estructura normal de una conexion. Esto puede ayudar a evitar la deteccion de los sistemas de seguridad, pero puede ser menos confiable en terminos de resultados.
  • Analisis de respuestas: A medida que se envian los paquetes y se reciben las respuestas, se analizan para determinar el estado de los puertos. Los puertos abiertos indican que hay servicios o aplicaciones en ejecucion en esos puertos, mientras que los puertos cerrados o filtrados indican que no se ha detectado ninguna respuesta o que se ha bloqueado el acceso.
  • Informe de resultados: Una vez completado el escaneo, se genera un informe que muestra los puertos encontrados abiertos, cerrados o filtrados en el dispositivo objetivo. Este informe puede ayudar a identificar vulnerabilidades, servicios expuestos o posibles riesgos de seguridad.

  • Ataques de denegacion de servicio (DoS): Estos ataques buscan abrumar un sistema o red con una gran cantidad de trafico o solicitudes, lo que provoca una interrupcion o una degradacion significativa del servicio. El objetivo es agotar los recursos disponibles, como el ancho de banda de la red o la capacidad de procesamiento del sistema, para hacer que el servicio sea inaccesible para los usuarios legitimos.
  • Ataques de fuerza bruta: Consisten en intentar adivinar contrasenas o claves de cifrado mediante la prueba sistematica de todas las posibles combinaciones. Los atacantes utilizan programas automatizados para realizar miles o incluso millones de intentos en un corto periodo de tiempo, con la esperanza de encontrar la combinacion correcta.
  • Ataques de phishing: Estos ataques se basan en la ingenieria social y buscan enganar a los usuarios para que revelen informacion confidencial, como contrasenas, numeros de tarjetas de credito o informacion personal. Los atacantes suelen enviar correos electronicos o mensajes falsos que se hacen pasar por entidades legitimas, como bancos o servicios en linea populares, y solicitan a los usuarios que proporcionen sus datos sensibles en sitios web falsificados.
  • Ataques de inyeccion de codigo: Los atacantes aprovechan las vulnerabilidades en las aplicaciones web para insertar y ejecutar codigo malicioso en el servidor o en el cliente. El ataque mas comun es el SQL injection, donde se insertan comandos SQL en formularios de entrada para manipular la base de datos subyacente.
  • Ataques de hombre en el medio (MitM): En este tipo de ataque, el atacante se situa entre el remitente y el destinatario legitimos y puede interceptar, modificar o incluso insertar comunicaciones sin ser detectado. Esto puede permitir al atacante robar informacion confidencial, como credenciales de inicio de sesion o datos financieros.
  • Ataques de spoofing: Los ataques de spoofing implican falsificar o suplantar la direccion IP, la direccion MAC u otras identidades en un intento de enganar a los sistemas y hacer que crean que el atacante es una fuente confiable. Esto puede permitir al atacante realizar actividades maliciosas, como robar informacion o realizar ataques de denegacion de servicio.
  • Ataques de malware: Los ataques de malware implican la entrega de software malicioso a traves de diferentes medios, como correos electronicos infectados, descargas no seguras o sitios web comprometidos. Los tipos comunes de malware incluyen virus, gusanos, troyanos y ransomware, que pueden danar o controlar el sistema afectado.

La vulnerabilidad IDOR ocurre cuando el identificador de un objeto interno, como un numero de ID o una URL, se utiliza directamente en la logica de la aplicacion o en la generacion de solicitudes, sin verificar si el usuario tiene los permisos adecuados para acceder a ese objeto en particular. Esto permite a un atacante manipular el identificador y acceder a objetos o datos de otros usuarios o de areas restringidas del sistema. Un ejemplo comun de una vulnerabilidad IDOR es cuando una aplicacion web utiliza el identificador de un objeto, como el numero de identificacion de un usuario, en la URL para acceder a su perfil. Si no se realiza una verificacion adecuada de los permisos, un atacante podria modificar el numero de identificacion en la URL y acceder a los perfiles de otros usuarios sin autorizacion. Las vulnerabilidades IDOR pueden tener consecuencias graves, ya que permiten el acceso no autorizado a informacion sensible, como datos personales, registros financieros o informacion confidencial de la empresa. Los atacantes pueden explotar esta vulnerabilidad para realizar acciones maliciosas, como robar datos, modificar registros, eliminar archivos o realizar acciones en nombre de otros usuarios legitimos. Para prevenir las vulnerabilidades IDOR, es importante seguir buenas practicas de seguridad, como implementar una adecuada autenticacion y autorizacion en el sistema, utilizar identificadores opacos en lugar de numeros secuenciales como identificadores de objetos y realizar verificaciones de permisos en cada acceso a objetos internos. Ademas, las pruebas de seguridad, como las pruebas de penetracion y el analisis de codigo, pueden ayudar a identificar y mitigar las vulnerabilidades IDOR antes de que sean explotadas por atacantes.

La autenticacion multifactor es un metodo de autenticacion que requiere que el usuario proporcione dos o mas formas diferentes de identificacion antes de acceder a un sistema o servicio. Normalmente, estas formas de identificacion incluyen una contrasena y algun tipo de factor adicional, como un codigo generado por una aplicacion de autenticacion, una huella dactilar o un escaneo de retina. Este metodo es mas seguro que la autenticacion de contrasena unica, ya que incluso si un atacante logra obtener la contrasena del usuario, aun necesitara el segundo factor para acceder a la cuenta.

El SQL injection es una tecnica de ataque comun en la cual un atacante inserta codigo SQL malicioso en una consulta SQL enviada a una aplicacion web, con el objetivo de manipular la base de datos subyacente. Esta vulnerabilidad ocurre cuando una aplicacion web no valida o sanitiza adecuadamente la entrada del usuario antes de construir consultas SQL dinamicamente.

Mediante el SQL injection, un atacante puede realizar diversas acciones, como obtener informacion confidencial de la base de datos, modificar o eliminar datos, o incluso ejecutar comandos arbitrarios en el sistema. Para prevenir el SQL injection, es fundamental utilizar consultas parametrizadas o preparadas, y validar y sanitizar correctamente la entrada del usuario antes de incorporarla en las consultas SQL.

El ataque XSS (Cross-Site Scripting) es una vulnerabilidad que permite a los atacantes insertar scripts maliciosos en paginas web vistas por otros usuarios. Estos scripts se ejecutan en el navegador de las victimas, lo que les permite robar informacion confidencial, redirigir a los usuarios a sitios falsos, modificar el contenido de la pagina y realizar otras acciones indeseadas.

El XSS ocurre cuando una aplicacion web no valida o escapa adecuadamente la entrada del usuario antes de mostrarla en las paginas web. Los atacantes aprovechan esta vulnerabilidad para insertar codigo malicioso, generalmente en forma de JavaScript, que se ejecuta en el contexto del navegador de la victima.

Para prevenir el XSS, es importante implementar medidas de seguridad, como validar y escapar correctamente la entrada del usuario, utilizar encabezados de seguridad HTTP adecuados y aplicar politicas de contenido seguro en el navegador. Tambien es recomendable utilizar frameworks y bibliotecas de desarrollo que incorporen mecanismos de proteccion contra XSS.

Para proteger una red Wi-Fi, se pueden implementar varias medidas de seguridad:

  • Utilizar un cifrado solido: Configurar la red Wi-Fi para utilizar el protocolo de cifrado WPA2 o WPA3 (Wi-Fi Protected Access) en lugar de WEP (Wired Equivalent Privacy), ya que este ultimo es menos seguro. Ademas, es importante establecer una contrasena segura y cambiarla regularmente.
  • Ocultar el nombre de la red (SSID): Desactivar la difusion del nombre de la red (SSID) para que no sea visible para dispositivos cercanos. Esto puede dificultar el acceso no autorizado a la red.
  • Filtrar direcciones MAC: Configurar el enrutador Wi-Fi para permitir unicamente las conexiones de dispositivos cuyas direcciones MAC esten en una lista de acceso autorizado. Esto ayuda a evitar que dispositivos no autorizados se conecten a la red.
  • Establecer niveles de acceso: Configurar diferentes niveles de acceso para los usuarios de la red Wi-Fi. Por ejemplo, se pueden crear redes separadas para invitados con acceso limitado a los recursos de la red principal.
  • Actualizar el firmware del enrutador: Mantener actualizado el firmware del enrutador Wi-Fi con las ultimas correcciones de seguridad proporcionadas por el fabricante.
  • Desactivar funciones innecesarias: Desactivar funciones de enrutador que no se utilizan, como UPnP (Universal Plug and Play) o WPS (Wi-Fi Protected Setup), ya que pueden presentar vulnerabilidades de seguridad.
  • Configurar un cortafuegos: Utilizar un cortafuegos (firewall) en el enrutador para controlar y filtrar el trafico de red entrante y saliente. Esto ayuda a proteger la red de posibles amenazas externas.

  • Educar a los usuarios: Brindar educacion y concientizacion sobre practicas de seguridad cibernetica a los usuarios de la red, como el uso de contrasenas seguras, evitar hacer clic en enlaces sospechosos y no compartir informacion confidencial a traves de la red Wi-Fi.

La ingenieria social es una tecnica utilizada por los hackers para manipular psicologicamente a las personas y obtener informacion confidencial o acceso no autorizado a sistemas. Algunas formas comunes de hackeo por ingenieria social incluyen:

  • Phishing: Los hackers envian correos electronicos o mensajes falsos que parecen legitimos, solicitando a las personas que revelen informacion confidencial como contrasenas, numeros de tarjetas de credito o datos personales. Estos mensajes suelen incluir enlaces a sitios web falsos que imitan a sitios autenticos.
  • Vishing: Similar al phishing, pero en lugar de utilizar correos electronicos, los hackers utilizan llamadas telefonicas. Se hacen pasar por representantes de empresas o instituciones legitimas y solicitan informacion confidencial a las victimas.
  • Ingenieria social presencial: Los hackers interactuan directamente con las personas en entornos fisicos, como oficinas o conferencias, para obtener informacion o acceso no autorizado. Pueden hacerse pasar por empleados, contratistas o tecnicos de soporte y utilizar tacticas persuasivas para enganar a las personas.
  • Pretexting: Consiste en inventar una historia ficticia para ganar la confianza de las personas y obtener informacion. Los hackers pueden hacerse pasar por empleados, autoridades o amigos para obtener datos confidenciales o acceso a sistemas.
  • Spear phishing: Es una forma mas dirigida de phishing, en la cual los hackers investigan y personalizan los mensajes de manera especifica para sus objetivos. Utilizan informacion obtenida de fuentes publicas o brechas de seguridad para aumentar la credibilidad del mensaje.

    • Para protegerse de los hackeos por ingenieria social, es importante mantenerse alerta y seguir buenas practicas de seguridad, como no revelar informacion confidencial a traves de medios no seguros, verificar la autenticidad de las solicitudes antes de responder y utilizar herramientas de seguridad, como filtros de correo electronico y programas antivirus, para detectar y bloquear intentos de phishing o malware.

El Credential Stuffing es una tecnica utilizada por los ciberdelincuentes para obtener acceso no autorizado a cuentas de usuarios mediante el uso de credenciales filtradas. Consiste en tomar combinaciones de nombres de usuario y contrasenas obtenidas de violaciones de datos previas y probarlas automaticamente en diversos servicios y plataformas, utilizando bots o programas automatizados.

    Dado que muchas personas tienden a reutilizar las mismas contrasenas en diferentes cuentas, los atacantes se aprovechan de esto para probar las credenciales filtradas en otros servicios populares, como redes sociales, plataformas de correo electronico o tiendas en linea. Si las credenciales funcionan, los ciberdelincuentes pueden acceder a la cuenta comprometida y realizar diversas acciones maliciosas, como robar informacion personal o financiera, enviar mensajes fraudulentos o realizar compras no autorizadas.

    Para protegerse del Credential Stuffing, es fundamental seguir buenas practicas de seguridad, como utilizar contrasenas unicas y robustas para cada cuenta, implementar la autenticacion de dos factores (2FA) cuando este disponible, monitorear las actividades de las cuentas y mantenerse informado sobre las ultimas violaciones de datos para tomar las medidas necesarias, como cambiar las contrasenas de las cuentas afectadas. Ademas, las organizaciones y proveedores de servicios tambien deben implementar medidas de seguridad, como la deteccion de patrones de acceso sospechosos y la limitacion de intentos de inicio de sesion fallidos, para mitigar los riesgos asociados con el Credential Stuffing.
